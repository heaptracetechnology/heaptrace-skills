Developer Pack

Core development skills — plan features, write code, debug issues, generate tests, craft commits, and ship releases.

13 Skills
feature-plan
Plan a new feature from a requirement or idea. Breaks it down into clear tasks, creates UI mockups, defines data flow, identifies risks, and produces a ready-to-implement plan.
developer / feature-plan
feature-work
Implement a feature end-to-end from a plan or requirement. Handles the full build cycle: setup branch, write code layer by layer (DB, Backend, Frontend), test, and verify.
developer / feature-work
find-fix
Investigate and fix a bug from a report, error message, or user complaint. Follows a structured flow: reproduce, trace, find root cause, fix, verify, prevent regression.
developer / find-fix
smart-commit
Analyze staged changes and generate a clear, semantic commit message that explains the WHY, not just the WHAT. Handles single commits, multi-file changes, and breaking changes.
developer / smart-commit
suggest
Read a task, requirement, or feature request and suggest improvements, missing pieces, edge cases, and value-adds the client did not ask for but clearly needs.
developer / suggest
code-review
Review code changes for logic errors, security issues, performance problems, duplication, and best practice violations. Use before merging a PR or after completing a feature.
developer / code-review
test-gen
Generate meaningful tests for a file, function, API endpoint, or feature. Covers unit tests, integration tests, and edge cases. Use after writing code to ensure it works correctly.
developer / test-gen
explain
Deep-dive explanation of any code, system, or feature. Breaks down how it works, why it was built that way, how data flows, what depends on it, and what would break if you changed it.
developer / explain
sec-audit
Scan code for security vulnerabilities — OWASP Top 10, exposed secrets, broken authentication, injection attacks, insecure dependencies, and missing protections.
developer / sec-audit
release-notes
Generate human-readable release notes from git history between two points. Groups changes by type (features, fixes, improvements), written in client-friendly language.
developer / release-notes
quick-plan
Lightweight planner for small tasks — bug fixes, small refactors, field additions. Produces a ~100-line plan instead of feature-plan's 500-line spec. Ends with A/B choice to save the plan or implement via quick-work.
developer / quick-plan
quick-work
Lightweight implementer paired with quick-plan. Reads the plan, executes each step, verifies acceptance, and offers A/B choice to commit or commit+push. Skips ceremony that small tasks do not need.
developer / quick-work
code-standards
Two-pass calibrate-then-audit code quality skill. Detects existing project conventions per language (TS, Python, Go, Java, etc.), then enforces them. Three modes (STRICT/DELTA/BASELINE) so legacy debt isn't punished. Test-maturity aware — never floods NO_TESTS projects with errors.
developer / code-standards
