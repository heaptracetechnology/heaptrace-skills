Lead Engineer Pack

Engineering leadership — sprint planning, architecture reviews, tech debt audits, incident response, and team onboarding.

8 Skills
sprint-plan
Break epics into sprints, estimate effort, assign tasks, identify blockers and dependencies. Use when starting a new sprint cycle, planning a large feature, or reorganizing the backlog.
lead-engineer / sprint-plan
arch-review
Review architecture decisions for scalability, coupling, single points of failure, and maintainability. Use before committing to a design or when refactoring a critical subsystem.
lead-engineer / arch-review
tech-debt-audit
Scan a codebase for technical debt — outdated patterns, missing tests, hardcoded values, dead code, inconsistent conventions, and security gaps.
lead-engineer / tech-debt-audit
incident-response
Structured incident handling — triage severity, identify root cause, implement fix, write postmortem. Use when production is broken or after an outage for retrospective analysis.
lead-engineer / incident-response
pr-strategy
Split large features into small, reviewable PRs with correct dependency order and feature flags. Use when planning a multi-day feature or when a PR has grown too large.
lead-engineer / pr-strategy
onboard-dev
Generate a comprehensive onboarding guide for a new developer joining the project. Covers architecture overview, local setup, codebase walkthrough, conventions, and key workflows.
lead-engineer / onboard-dev
perf-audit
Profile slow endpoints and pages — find N+1 queries, missing indexes, large bundle sizes, memory leaks, and unnecessary re-renders. Use when response times are slow.
lead-engineer / perf-audit
decision-doc
Write an ADR (Architecture Decision Record) — document context, options considered, decision made, trade-offs, and consequences. Use when making significant technical choices.
lead-engineer / decision-doc
