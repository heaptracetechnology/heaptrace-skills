Architect Pack

System architecture — API design, database modeling, integration planning, migration strategies, and scalability reviews.

8 Skills
system-design
Design a system from scratch — components, data flow, API contracts, scaling strategy. Use when starting a new product, service, or major feature that needs architectural planning.
architect / system-design
api-design
Design RESTful APIs — endpoints, request/response schemas, error codes, versioning. Use when planning new API endpoints, redesigning existing APIs, or creating API contracts.
architect / api-design
db-design
Design database schema — tables, relations, indexes, migration strategy. Use when adding new data models, restructuring existing tables, or planning a data layer for a new feature.
architect / db-design
integration-plan
Plan third-party integrations — OAuth, webhooks, rate limits, error handling. Use when integrating with external services like Stripe, Google, Slack, or any API-based service.
architect / integration-plan
migration-plan
Plan data and system migrations — zero-downtime strategy, rollback procedures, validation checks. Use when migrating databases, switching providers, or restructuring schemas.
architect / migration-plan
scalability-review
Review a system for scaling bottlenecks — caching, queues, read replicas, CDN, load balancing. Use when preparing for traffic growth or diagnosing performance issues.
architect / scalability-review
event-design
Design event-driven architecture — event schemas, pub/sub patterns, idempotency, ordering guarantees. Use when decoupling services, building notification systems, or audit trails.
architect / event-design
cost-estimate
Estimate infrastructure cost for a feature or system — compute, storage, bandwidth, managed services. Use when planning budgets, comparing architectures, or evaluating build-vs-buy.
